# Hands-on: Build the Space

You fine-tuned a model in Unit 1. Time to give it a demo people can actually click on.

## The Setup

Use the starter that ships with this unit:

```bash
cp -R course/units/en/unit2/starter ~/unit2-starter
cd ~/unit2-starter
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
mkdir -p .claude/skills
cp -r skills/hf-brand .claude/skills/
cp -r skills/brutalist .claude/skills/
```

> [!NOTE]
> If you did not complete the Unit 1 fine-tune, that is fine. The starter's `app.py` uses a mock `predict()` function so you can still do the whole exercise.

## What the Brand Skill Does

Open `.claude/skills/hf-brand/SKILL.md`. It encodes a specific look and tone for Gradio apps:

- Primary yellow `#FFD21E`, orange `#FF9D00` hover
- The 🤗 emoji as the brand mark
- Source Sans Pro font, soft rounded corners
- Warm, welcoming copy — "Try it out!" not "ENTER TEXT"
- A ready-to-use Gradio theme block

It is a brand guideline in skill form.

## Build It

Open Claude Code in `~/unit2-starter`:

```bash
claude
```

Ask:

```text
Turn `app.py` into a Space-ready Gradio demo for the sentiment classifier.
Keep the mock `predict()` function. Add three example inputs and a polished
layout, but do not change the underlying prediction logic yet.
```

You never said "use the hf-brand skill." Claude should infer that from the request.
